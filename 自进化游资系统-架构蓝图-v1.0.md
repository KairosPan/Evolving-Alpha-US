## 0. 一页纸总览(TL;DR)

- **环境 environment** = A 股超短市场(**非平稳 non-stationary + 对抗 adversarial + 反身 reflexive**)。
- **观测 `o_t`** = 每日(及盘中)市场状态:情绪值、连板梯队、赚钱/亏钱效应、指数结构、流动性、**per-题材线** 的相位向量、个股盘口。
- **动作 `a_t`** = 一个 **`DecisionPackage` 决策包**(排序候选池 + 每只的模式/买点/卖点/**仓位档** + 理由 + 禁忌检查 + 不参与理由),**人类游资确认下单**。
- **Harness `H=(p,G,K,M)`**:`p`=当前作战 **doctrine**;`G`=游资子 Agent 群;`K`=葵花宝典 + 模式/特征**技能库**;`M`=**复盘知识库**。四者通过 meta-tool API 被就地 CRUD 编辑。
- **Inner loop(每日收盘复盘,reset-free)**:Refiner 读最近 trajectory → 识别失败签名 → 对 p/G/K/M **四遍各一遍** CRUD。一个 **episode = 一个完整周期轮回**(冰点→修复→主升→震荡→退潮→冰点)。
- **Outer loop(跨周期协同学习)**:本地策略模型 `π_θ` 在历史回放跑 rollout → **PRM** 过程奖励 → 低奖励窗口由 **frontier teacher + 已实现未来 oracle** relabel → **soft-SFT/LoRA** 更新 θ,reset-free 历史遍历。
- **推荐**:**方案 A(直译)= 目标态;方案 B(Harness-only 冻结模型)= MVP/Phase-1;方案 C(锦标赛/进化算法)被吸收为 K/G 的孵化—选择机制(育种场)**。
- **交易 vs Pokémon 的本质差异(本蓝图最核心的工程主张)**:交易多一个**天然完美 oracle = 已实现的未来**(↔ Dijkstra),但同时打破了论文的两条隐含前提——**oracle 会漂移(alpha decay)** 且 **环境对抗反身**。因此在论文双环之上,强制叠加**非平稳六道闸**:regime 标签 + decay 加权 + dormant 技能"轮回"复活 + alpha-decay 监控 + 能力地板回退 + OOS/影子盘/PIT 防前视。

## 1. 为什么是 Continual Harness(核心论点)

`轮回.docx` 自己反复给出一条元论点:

> "市场在高度进化……不断进化,由前人经验,不断完善。"(/tmp/lunhui.txt:1569–1580 创业板打法进化史;:1695 生态轮转)

它讲的是:任何被写下来的打法,一旦被足够多人套利,溢价就消失甚至反转(alpha decay)。书名"轮回"= 生态/题材/打法的循环往复——旧核止跌、新核反核,次新周期与超跌周期互为生死(:1704–1708)。

**这正是 Continual Harness 存在的理由。** 论文要解决的是:embodied、长程、部分可观测的决策任务里,没有一套固定 scaffolding 能一劳永逸;唯一可持续的是一个**会自己改 scaffolding 的 harness**。把这句话翻译到交易:

> **可持续的不是某一个打法,而是一套会自己改打法的系统。**

二者还在更深的结构上同构:

| 论文(Continual Harness) | 轮回(游资) |
|---|---|
| reset-free episode,失败签名跨步单调累积、精炼质量随回合长度复利 | 一个完整周期轮回内,越到退潮越要靠前面攒下的 doctrine;周期不重启 |
| 能针对只在回合深处出现的失败(后期战斗/多步谜题) | 能针对只在周期深处出现的失败(退潮末/二波/补涨尾声) |
| 平稳环境 + 完美固定 oracle(Dijkstra) | **非平稳 + 对抗**,但有**已实现未来**这一近完美却**会漂移**的 oracle |

第三行的差异是本蓝图相对论文的全部技术张力所在,§8 专论。

## 2. 三个候选顶层架构方案与推荐

### 2.1 方案 A —— "Continual Harness 直译"(单一自精炼 harness + 协同学习两环)

1:1 复刻论文。维护**单一活体** harness `H=(p,G,K,M)`;inner loop 每日/每相位由 Refiner 自精炼 H(CRUD);outer loop 跨周期用 DAgger+PRM+teacher relabel 更新本地策略模型 θ。回放优先,**已实现未来**当 oracle。四层全进化。

| 维度 | 评价 |
|---|---|
| 论文忠实度 | ★★★★★ 完全对齐双环 + reset-free 复利 |
| in-episode 复利 | ★★★★★ 周期轮回内 doctrine 累积,能修周期深处才出现的失败 |
| 领域契合 | ★★★★★ "轮回"=reset-free reincarnation,概念同构 |
| 工程成本 | ★★★☆☆ 需 PRM + teacher + LoRA + GPU,数据管线重 |
| 风险 | 单一 H 可能 curve-fit 到当前 regime;能力地板以下过拟合噪声(§8 防护) |
| 多样性 | ★★★☆☆ 单一谱系(由吸收 C 补) |

### 2.2 方案 B —— "Harness-only,模型冻结"(只在线进化 H)

砍掉 outer loop。基础模型权重冻结(用一个够强的本地大模型或 API),**只跑 inner loop**:Refiner 在线 CRUD 编辑 `p,G,K,M`。等价于论文里 "frozen base model + meta-harness" 的纯 in-context 自进化。

| 维度 | 评价 |
|---|---|
| 工程成本 | ★★★★★ 无 GPU/训练管线/PRM/teacher,纯 prompt+检索+技能注册表 |
| 上线速度 | ★★★★★ 数周出 MVP,先验证"H 自精炼是否真带 alpha" |
| 能力上限 | ★★★☆☆ 受冻结模型能力地板限制;K/M 增长但模型"读盘直觉"不进化 |
| 风险 | 低(无权重漂移);最坏退化成一个会自我整理的专家系统 |
| 与 A 关系 | A 的**真子集**;B 跑稳后**原地接上** outer loop 即升级为 A |

### 2.3 方案 C —— "多策略锦标赛 / 进化算法"(策略-Agent 种群)

维护策略-Agent **种群**(N 个 harness 变体 / 模式组合个体);历史回放上跑**锦标赛**,按风险调整绩效做 select/mutate/crossover,promote 赢家、淘汰输家。

| 维度 | 评价 |
|---|---|
| 多样性/鲁棒 | ★★★★★ 种群天然对冲单一 regime curve-fit |
| 探索 | ★★★★★ 变异/交叉发现非显然模式组合 |
| **in-episode 复利** | ★☆☆☆☆ **丢失** reset-free 周期内单调累积(个体被批量评估而非连续精炼) |
| 信用分配 | ★★☆☆☆ 只有种群级适应度,粒度粗 |
| 算力 | ★★☆☆☆ N 倍 rollout |

### 2.4 ⭐ 推荐与边界

> **方案 A 为目标态;把 C 的 tournament 吸收为 K/G 的孵化—选择机制(育种场);把 B 作为 MVP/Phase-1。**

- **为什么是 A**:① 概念近零阻抗同构(§1);② 交易有论文求之不得的完美 oracle = 已实现未来;③ 四层全进化是用户硬约束,只有 A 同时进化 p/G/K/M + θ。
- **吸收 C 为育种场**:Refiner 在 inner loop `create` 新技能/子 Agent 时**不直接上线**,先入**孵化锦标赛沙盒**,在历史回放 + 近期 OOS 上与现役版本竞争;变异(调买卖点阈值)、交叉(合并两个补涨模式);**OOS 上风险调整绩效显著胜出**才 promote 进现役。既保留 A 的 reset-free 复利(主干仍是单一活体 H),又用 C 的种群压力防单一谱系脆弱。
- **B 为 MVP**:先用最便宜方式证明"harness 自精炼在游资场景成立",再决定是否上权重进化。**强烈建议从 B 起步**(见 §9 roadmap)。
- **边界(不做什么)**:不自动下真单(只到决策包,人确认);能力地板以下不开 outer loop(§8.4);永远以 OOS 门 + 影子盘为准,不信训练曲线(论文 outer loop 未确立收敛点)。

```
                ▼

        └───────────────────┘                                       └──────┬───────┘
                                                                            │ τ 轨迹累积
   ┌────────────────────────────────────────────────────────────────────────┘
   ▼
┌──────────────────────────────────────────────────────────────────────────────────────┐
│ Harness 自进化引擎:  M 记忆/复盘 + Refiner(inner loop)  ⟂  协同学习(outer loop)       │
│   inner: 每日复盘 → 失败签名 → CRUD(Δp,ΔG,ΔK,ΔM)                                        │
│   outer: 回放 rollout → PRM → teacher+oracle relabel → LoRA θ                            │
```

**关键数据流原则(全局):**
1. **`H` 与 `s_t` 严格分离**(论文铁律,§6.2):`s_t` 是来自市场的客观观测,`H` 是 scaffolding。**禁止 harness 自己定义自己的观测**,否则陷入论文 B.3 "Feedback Blindness" 式自欺。
2. **未来函数防火墙**(§6.1/§6.6):实盘推理路径(L0–L4)只用 `≤t` 信息;"已实现未来"oracle **只在 outer-loop 训练的 relabel 环节出现,产物只进训练集,永不进实盘推理**。
3. **per-题材线并行**:不存在单一全局标量 regime;相位是题材线向量 + 一个全局母状态。

## 6. 子系统详细设计

> 每个子系统给出:**职责边界 · 输入/输出 · 关键数据 schema · 自进化机制(这一层如何被 Refiner/协同学习改写)· 依赖 · 交易特有坑与对策**。对抗评审的 P0/P1/P2 修正已逐条纳入并标注 `[修P#]`。

### 6.1 L0 数据与特征层(环境观测 `o_t`)

**职责**:把原始市场数据,转成模型可读的、**当时点可得的(point-in-time)** 市场状态向量;并把"特征计算"本身做成可自进化的技能。

**数据源**:行情 L1/L2 + 逐笔成交、龙虎榜(机构席位/知名游资/砸盘王标签)、板块/题材成分(及其历史时点版本)、公告(减持/重组/辟谣/问询)、资金流、转债、新股/次新、指数。
**节奏**:日频为主轴;盘中关键节点(竞价 9:15–9:25、连续竞价、尾盘 14:30+、临停)按事件触发。

**核心特征(把轮回状态变量做成可计算信号)**:
- **情绪值** `[修P3]`:**不用绝对阈值**(轮回原文的"冰点1500/高潮3500/4000顶值"会随市场容量漂移失效)。改为 **regime-relative 归一化**:对"昨日涨停今表现、连板溢价、炸板率、赚钱效应"等做滚动分位 / z-score;阈值本身作为可进化技能进 `K`。
- 涨跌家数、连板高度梯队 `[(高度,数量,代表票)]`、赚钱效应/亏钱效应强度。
- **炸板率**(>40% 次日接力风险小 / <20% 风险大,:1323)、昨日涨停/连板/炸板次日表现。
- 880005 五日线相位、>15 亿成交额个股占全 A 成交占比(容量行情指标,越近 30% 越危险,:1327)。
- **弱转强信号**(今日上板时间 vs 昨日,:773)、预期链档位映射(核按钮…百万封单一字,:1637)、梯队结构、龙虎榜标签。
- **题材轮动强度**(每条题材线 +1…−1,:1359)、**拥挤度/反身性特征**(同模式同现标的数、一字溢价收敛速度)→ 喂给 alpha-decay 监控(§8.2)。

**输出 schema `MarketState` `o_t`**:
```jsonc
{
  "date": "2026-05-29", "as_of": "15:00",            // as-of-date 时间戳,杜绝未来函数
  "sentiment_score_norm": 0.18,                       // regime-relative 归一化情绪
  "echelon": [{"height":7,"count":1,"code":"...","name":"..."}, ...],
  "money_effect": {"earn":0.3,"lose":0.7},
  "index": {"k_structure":"箱体下沿","macd_div":"30F顶背离","level":"60F"},
  "liquidity": {"two_market_turnover":..., "capacity_phase":"容量主升"},
  "themes": [{"theme":"可控核聚变","line_id":"T17","rotation":+1,"phase":"主升"},
             {"theme":"机器人","line_id":"T03","rotation":-0.5,"phase":"退潮"}],  // per-题材线
  "ecology_tags": ["连板生态","20cm生态"],
  "crowding": {...},
  "tradeable_universe_snapshot_id": "pit_20260529",   // 当时可交易池指针(PIT)
  "stock_snapshots": { "300xxx": {盘口,竞价,分时,封单,涨跌停状态,停牌?} }

```
`m_t`(notepad / 工作记忆读出)由 `M` 提供(§6.5),与 `o_t` 一起构成 `s_t=(o_t,m_t)`。

**自进化机制**:特征即 `K` 里的"特征技能"。Refiner 可 `write_skill` 新特征(如某 regime 下新发现的有效信号)、`retire_skill` 失效特征;新特征须过孵化锦标赛(§6.3)。

**交易特有坑与对策**:
- `[修P4]` **幸存者偏差 + 未来函数 = 头号杀手**:强制 **point-in-time 数据库**——当时可交易池、ST/退市/停牌、当时的代码↔名称映射(轮回的"代码联想 603333–603332":1398、"字辈":1392 都依赖历史时点名称);任何特征只用 `as_of` 之前可得信息;复权一致。
- 涨跌停/停牌作为**可执行性状态字段**进入快照(供 §6.7 fill-feasibility 用)。
- L2 数据缺失期 → 该 regime 标"数据地板",触发 §8.4 自进化冻结。

### 6.2 L1 周期/情绪状态分类器(子 Agent `G_cycle`)

**职责**:把轮回的"大周期(赚钱/亏钱效应+连板高度)+ 情绪周期(混沌冰点→修复→回暖→启动→主升→震荡补涨→退潮)"形式化为**可观测状态机**,输出当前 regime。

`[修P1+P2]` **三处关键重构(解决最严重内部矛盾)**:
1. **单一真相源 SSOT**:相位/情绪**属于 `s_t` 侧的客观标签,不属于 `H`**。`G_cycle` **只做"读取/预测",绝不把相位写回去定义自己的输入**。论文里 `H` 与 `s_t` 严格分离;naive 设计把相位塞进 `p`,会形成"Refiner 用相位触发→精炼重写含相位的 p→p 又喂 G 判相位"的 confirmation-bias 闭环。
2. **per-题材线向量,非单一标量**:轮回明说"一根 K 线可以是分歧和分歧延续态的结合"(:332)、"总龙被按、题材未到退潮、新龙出"(:277)——退潮与主升在不同题材线上**并存**。故输出 = **每条题材线一个相位** + **一个全局"流动性/赚钱效应"母状态**。
3. **可被 oracle 事后审计**:每个相位判定都用客观判据(连板高度、炸板率、880005 五日线、>15 亿成交额占比),事后用"已实现未来"验证"这天到底是不是退潮起点"。

**状态机(全局母状态 × 每题材线子状态)**:

| 状态(情绪相位) | you-see 可观测判据 | 转移信号(→下一态) |
|---|---|---|
| 混沌/冰点 | 老题材振荡、低位板块龙头先动、连板高度≤3、三连阴末阴 | 旧核大范围止跌 + 新核大范围反核 → 修复 |
| 修复启动 | 首板/换手 2 板辨识度个股、地天板反核 | 题材线显现、涨停潮 → 回暖/启动 |
| 情绪回暖 | 代表性换手 2 板、与指数共振、小范围涨停潮 | 3/4 板龙头确认主线 → 主升 |
| 题材启动日 | 板块大面积涨停(数量越多周期级别越大) | 一字龙竞价封单>10 亿 → 主升;集体断板留活口 → 启动失败 |
| 主升 | 龙头不断突破、脱离板块愈走愈强、涨停潮 | 第一根强分歧阴 K + 次日非强修复 → 震荡 |
| 震荡/补涨 | 亏钱效应扩散、个股淘汰赛、板块横盘龙头振荡 | 龙头破位放量大阴/缩量跌停 → 退潮;首阴反包 → 行情继续 |
| 退潮(退一→反弹→退二) | 龙头与补涨龙**共振下跌**、最高标被核、顶一字弱转强卡高度成大面之源 | 板高降至 4B−、旧核止跌新核反核 → 新冰点/新周期 |

**输入**:`MarketState o_t`(§6.1)。**输出**:
```jsonc
{ "global_mother_state": {"liquidity":"容量主升","money_effect_phase":"赚钱→亏钱过渡"},
  "lines": [{"line_id":"T17","phase":"主升","confidence":0.7,"sub_cycle_level":"中"},
            {"line_id":"T03","phase":"退潮","confidence":0.85}],
  "uncertainty": "T17 主升/震荡边界态" }
```
与**指数周期 / 流动性周期 / 生态周期(开拓者)** 多级嵌套(:1651–1708)。

**自进化机制**:`G_cycle` 的**判据**是可被 Refiner `patch_subagent` 编辑的;outer loop 用**已实现未来**做监督(事后这天到底是不是退潮起点)持续校准 judges。它是**高频精炼热点**之一(§6.5 master 分派)。它的输出**驱动葵花宝典**:regime → 可用模式集。

**坑与对策**:① 边界态硬切会误判 → 输出连续置信度而非硬标签;② 情绪绝对阈值漂移 → §6.1 已 regime-relative 归一化;③ 能力地板下分类器过拟合噪声 → §8.4 冻结。

### 6.3 L1/核心 策略模式库 `K`(葵花宝典 + 接力/龙头/补涨/套利 → 参数化技能)

**职责**:系统核心。把轮回所有可操作模式统一为**参数化技能对象**,带滚动绩效,可被 CRUD,**朝 oracle 自改进**(直接对应论文 4.6"技能朝 Dijkstra oracle 自改进")。

**技能对象 schema**:
```jsonc
{
  "skill_id": "W2S_2to3",                     // 例:二进三弱转强
  "type": "pattern",                           // pattern | feature | failure_detector
  "applicable_regime": ["修复","回暖"],         // 适用相位(per-题材线)
  "trigger": "二板放量封死/烂板; 次日竞价昨成比>5%且高开拉回示强",
  "entry": "竞价符合则扫; 否则板上确认/T字快速回封",
  "exit": "次日不及预期(顶值开盘)止损; 加速大分歧A杀",
  "size_hint": "中仓; 有竞争择强、看不清买少",
  "taboo": ["退潮期不做","板块跟风后排2B不做","庄股不做"],
  "depends_on": ["弱转强判定","预期链档位","梯队结构"],
  "examples": ["天顺股份2022/9/5 2-3","摩恩电气2018/11/21 2-3"],
  "stats": { "winrate": {"value":0.58,"decay_weighted":true,"by_regime":{...}},
             "pnl_ratio": 2.1, "expectancy": 0.012, "n": 47,
             "oracle_gap": 0.18 },                // ★交易版 path-cost deficit, 见下
  "confidence": 0.62,
  "status": "active"                              // active | incubating | retired | dormant
}
```

**`[修P11]` 交易版 oracle 度量(对应论文 Fig8 path-cost deficit)**:仅"滚动胜率"是**终任务度量**,不是论文要的**独立 oracle 度量**。补一个 **`oracle_gap`**:用已实现未来,事后重算"该技能在该 regime 下的理论最优买卖点",技能实际买卖点与之的差距(滑点/收益差)。这给出独立于 P&L 的技能自改进曲线——Refiner 在同周期内"早期调用失败→诊断→在后续调用前修复",`oracle_gap` 随回合复利收窄(in-loop & reset-free)。

**CRUD 生命周期**:
- `write_skill`:从成功序列 codify 新技能;
- `patch_skill`:修复"抛异常"(逻辑过期/失效)的技能(`oracle_gap` 上行触发);
- `retire_skill`:长期负期望 → 退役**但不删**;
- `revive_skill` `[修P9]`:退役技能进 **dormant 池**(降权但保留 regime 指纹);当 `G_cycle` 检测到"曾适配该技能的 regime 回归"→ 自动 revive 候选,重新进孵化锦标赛竞争上岗。**这正应"轮回"之名,是交易优于 Pokémon 之处**(废弃迷宫路径不会复活,退役打法会随生态轮转复活:旧核止跌新核反核)。

**`[修C]` 孵化锦标赛(吸收方案 C,防过拟合)**:新建/复活技能先入 `incubating` 沙盒,在历史回放 + 近期 OOS 上与现役版本竞争(变异调阈值、交叉合并模式);**OOS 风险调整绩效显著胜出才 promote 为 active**。

**`[修P19]` 把轮回"防坑结构"codify 为可执行失败检测技能**(`type:failure_detector`):黄昏之星、三柱香、双飞乌鸦、塔形顶、三只乌鸦、吊颈线、五阴线(:1466–1498)、出货分时(:1090)。这是 substrate 白送的失败签名种子,直接进 `K` 供 §6.7 风控调用。

**`[修P15]` 涌现技能审计闸**:涌现是双面的——好的=自发发现新套利结构(容量中军、300 弹性:1696);坏的=**自发发现回测漏洞/未来函数 loophole 并封装成"技能"**(=论文 sandbox loophole,在交易里就是作弊)。**每个 create/promote 的技能必须过审计闸**:检查它是否隐式依赖了 `as_of` 之后的信息、是否在模拟成本下才成立(模拟成本 vs 实盘成本 diff 异常 → 拦截,§6.6/§8.5)。

**坑与对策**:① 多技能同标的命中冲突 → 按 `confidence × oracle_gap × regime 适配度` 加权仲裁,同向加分、反向取高置信;② 过拟合历史范例 → 孵化必带 OOS 门 + survivorship 负例;③ 把游资玄学天真量化丢 nuance → 模式技能保留"理由/禁忌"文本字段供模型推理,不只给数值阈值。

### 6.4 L2 选股/排序子 Agent + 决策编排器(产出 Co-pilot 决策包)

**职责**:= 论文 orchestrator(Agent 读 `(s_t,H_t,τ)` 产 `a_t`)。日内把感知→匹配→排序→决策包串起来,产出给人确认的结构化输出。

**`[修P12]` master_龙头_agent 分派结构(对应论文 master_battle_agent)**:不把所有读盘逻辑摊平,而是建**主控分派 Agent**,把 per-decision 逻辑分派给具名 sub-check:`弱转强检查 / 卡位检查 / 补涨检查 / 退潮接力否决检查 / 预期差档位检查`(正对应轮回 :654–1290 的模式分类)。其中**弱转强 / 退潮接力否决 / 预期差**被指定为**高频精炼热点**(交易里最易 decay 的判定,§6.5)。

**日内时序**:
| 节点 | 动作 |
|---|---|
| 盘前 | 读 regime(§6.2)→ 选可用技能集 → 建候选池(自选板块+消息+昨日异动) |
| 竞价 9:15–9:25 | **预期差判读**(预期链档位 vs 合理预期,:1637)→ 排序;9:20 前撤单观察 |
| 盘中 | 模式触发实时更新候选;支线切换/板块轮动跟踪 |
| 尾盘 14:30+ | 炸板低吸/地天板/接力/明日预期 |
| 收盘 | 触发 inner-loop 复盘(§6.5) |

**`[修P5]` 决策包 `DecisionPackage` `a_t` schema(人机界面)**:
```jsonc
{
  "date":"...", "regime_read":{...}, "sentiment_phase":"...",
  "ranked_candidates":[
    {"code":"...","name":"...","pattern":"二进三弱转强","skill_id":"W2S_2to3",
     "entry":"竞价昨成比>5%高开拉回扫","exit_stop":"顶值开盘止损",
     "size_tier":"中(建议≤X%)","exp_gap_ladder":"小阴→大额一字(超预期)",
     "fill_feasibility":{"buyable":true,"seal_amount":"8亿","queue_risk":"中"},  // [修P5]
     "taboo_check":[{"rule":"退潮不接力","status":"过"},{"rule":"庄股","status":"过"}],
     "reason":"引用 K 技能 + M 历史类比(摩恩电气2-3)","confidence":0.62,

  "no_trade_reason":"指数预期回落下午不参与分歧低吸",
  "key_risks":["神马电力式坏头出现则全面退潮"],
  "human_confirm": null            // 人填:confirm | reject | modify(+理由)

```

**`[修P6]` 人类确认回流 = DAgger 专家标注**:人类 confirm/reject/modify **及其理由**是高价值的 on-policy 专家纠正,直接作为 DAgger 数据进 outer loop(这是 co-pilot 模式相对全自动的独特优势:免费拿到专家标注)。

**自进化机制**:编排策略(候选池构建、排序权重、master 分派逻辑)由 `π_θ`(outer loop 微调)+ `p` doctrine(inner loop 重写)共同决定;Agent 与 Refiner 共享同一模型 `M`。

**坑与对策**:① 人机摩擦 → 决策包默认按置信度折叠,一屏可决;② 候选池爆炸 → 先按 regime 可用技能集硬过滤;③ 把不可成交标的排前 → fill-feasibility 字段必填且参与排序(§6.7)。

### 6.5 记忆 `M` + Refiner 内环(每日/每周期复盘 → CRUD 自精炼)

**职责**:= 论文两环里的 inner loop(reset-free 在线精炼)。记忆是知识沉淀,Refiner 是把轨迹变成 `H` 编辑的引擎。

**`[修P9]` 记忆 `M` schema(放弃"单调累积",改 regime 时戳 + decay)**:

长期记忆(教训/类比/失败签名)+ 工作记忆(`m_t`,当日盘面便签)。**旧 regime 条目降权不删**,留待"轮回"复活。

**Refiner 触发与四遍(忠实论文 §3.2)**:F = 1 交易日收盘复盘 + 事件触发(相位切换大复盘);W = 上线预热期只观测不编辑(防早期噪声污染 H)。Refiner 读 `τ_{t-F:t}`,识别**交易失败签名**:追高被套 / 该走不走(盛极转衰非龙头早退龙头迟退违例)/ 卡位失败 / 退潮期接力 / 一进二顶一字买不进 / 预期差 read 反了(以为修复实为退潮)/ 把杂毛当龙头 / 被家人榜迷惑 / 踩防坑结构。然后**四遍各一遍 CRUD**:
1. **Δp**:重写 doctrine(当前 regime 作战指导 + 激活的禁忌口诀);周期性"增长↔简化",必要时结构重写(把零散逻辑吸收进 master 分派子 Agent)。
2. **ΔG**:对重复多步读盘模式 `create` 子 Agent;对检测到失败的识别器 `edit`;对从没 productively 调用的 `delete`。
3. **ΔK**:从成功序列 codify 技能、修复 `oracle_gap` 上行的技能、retire 负期望、revive dormant(§6.3)。
4. **ΔM**:补记忆空白、更新陈旧条目、对已越过 regime 的条目 demote 重要度。

**`[修P18]` 精炼信息"会过期",不是单调累积**:论文 reset-free 单调累积成立**因为 Pokémon 平稳**;交易非平稳,打法会被套利掉失效(:1569–1580)。因此 inner-loop 内**同 episode、同 regime**信息累积有效(局部单调),**跨 regime 则按 decay 加权**,且保留 dormant 复活通道。**这是本蓝图对论文最重要的一处修正。**

**`[修P3 不可侵犯核]`**:保留一个不可变的核心 doctrine(纪律/止损/风控红线),Refiner **不能**改写,防灾难性遗忘。

**Worked example(用轮回真实退潮案例,:437–441)**:
> 6/27 集泰、世宝高位断板后强势反包(情绪好)→ 6/28 神马电力率先走弱断板、当天大阴 → 6/29 暴跌跌停,高位人气股大面积 A 杀。
- **失败签名**:若系统在 6/28 仍按"修复"接力 → 标记"退潮接力 + 预期差 read 反了"。
- **Δp**:doctrine 加"出现率先走弱的最高连板(神马式坏头)→ 立即下调全市场溢价预期,回避高位股"。
- **ΔG**:`patch` `G_cycle` 退潮判据(把"最高连板率先走弱断板+大阴"提为强退潮信号)。
- **ΔK**:把"连续退潮不做高位反包博弈"硬化进相关接力技能的 taboo;相关技能 `oracle_gap` 上行 → 降权。
- **ΔM**:写入具名类比"神马电力2024/6/28",regime=退潮拐点。

**坑与对策**:① 把噪声/运气当教训 → edit 须过统计显著性 + OOS 门控(§8.5);② 灾难性遗忘 → 不可变核 + diff 可回滚;③ Refiner 自身能力地板 → §8.4。

### 6.6 协同学习外环(回放环境 + PRM + teacher relabel + soft-SFT)

**职责**:= 论文 Fig2b outer loop。用历史回放把本地策略模型 `π_θ` 训起来,与 H 协同适配。

1. **回放环境(reset-free emulator)**:多年 A 股日线/盘中做成可遍历 emulator,一次迭代 = K 个交易日决策窗口(K≈256);**迭代 k 末的回放游标 + 模拟持仓态 = k+1 起点**(reset-free 历史遍历)。强制 as-of-date 快照。
2. **策略模型 `π_θ`** = 本地/开源模型(对应论文 Gemma),在**活精炼的 H** 内跑 rollout。
3. **`[修P4]` PRM 过程奖励(未来函数防火墙是头号铁律)**:
   - 分解(对应论文 progress/action-correctness/reasoning/format):**regime-read 正确性 · 模式判定正确性 · 纪律遵守(退潮不接力等)· 预期差兑现度 · 风险调整终端收益(扣滑点/涨停买不进/流动性)**。
   - **铁律**:PRM 在 `t` 时刻**只能用 `≤t` 信息**;"弱转强是否兑现""预期差是否兑现"这类需要 `t+1` 的判断,**只允许出现在 teacher relabel 的监督信号里,绝不进实盘推理特征**。relabel 产物**只进训练集,永不进推理路径**。
4. **`[修P17]` teacher 与 oracle 职责分离**:
   - **已实现未来 oracle** 给**客观结果标签**(该买谁/该走/该空仓/最优买卖点)——这是交易相对 Pokémon 的核心优势。
   - **frontier teacher(强 LLM)** 只负责把 oracle 的结果**归因成"决策包 + 理由"**(因果解释),供 soft-SFT。**二者不能混**:否则若 teacher 的游资知识本身来自轮回.docx,会造成 teacher=substrate=被评估对象的**循环验证**。
5. **soft-SFT / LoRA** 在 relabeled shard 上轻量更新 θ(~3ep, ~5e-6);warmup = SFT on frontier trajectories + 离线 GRPO on 过程奖励(单独 warmup 不产生里程碑推进,增益始于 co-learning,忠实论文 §4.5)。
6. **θ 与 H 协同**:模型动作诱导 `τ` → Refiner 读 `τ` 更新 `H` → `H` 塑造下个观测分布。θ 跨迭代更新,H 回合内更新。
7. **`[修P16]` θ 版本管理 / 影子上线 / 回滚**:论文诚实承认 outer loop 未确立收敛、PRM reward 非单调、resume 后回归。交易非平稳,θ 跨周期 SFT 累积 = 持续往过去 regime 拟合。故:θ 全程版本化;新 θ 先**影子盘 A/B**;**OOS 不显著优于 θ_k 即回滚**;设停训准则(连续 N 迭代 OOS 无提升)。

**坑与对策**:① 未来函数泄漏(最危险)→ §6.6 铁律 + 回放引擎硬隔离;② 过拟合历史 regime → OOS 门 + frozen 对照(§8);③ 模拟成本失真 → `[修P6]` **模拟成本 vs 实盘已实现成本 diff 作为过拟合警报**;④ 能力地板 → §8.4。

### 6.7 L3+L4 仓位/组合层 + 风控守卫(含止损器、监管否决、fill-feasibility)

**`[修P9-仓位]` 仓位/组合/相关性层(naive 版完全缺失的关键一层)**:轮回反复讲"满仓发酵期"(:188)、"控仓参与"(:1286)、"空仓修身"(:12)、分批。Co-pilot 只给"排序候选池"而不给仓位 = 残缺,且**仓位比单股选择更决定实盘存活**。
- 输入:决策包候选 + 各自 confidence + regime + 现有持仓。
- 输出:每只**仓位档**(空仓/试仓/中仓/重仓)+ 风险预算(单票/单日上限)+ **相关性约束(同题材多票 = 单一押注,合并计风险敞口)** + 分批节奏。
- 自进化:仓位映射(confidence→仓位档、regime→总仓位)作为技能进 `K`,由 oracle(已实现波动/回撤)校准。

**`[修P10]` 风控/止损器 `G_risk`(机制规格,非只挂名)**:止损是**情绪态条件式**的多触发器:
- **形态止损**:个股分时破位/防坑结构命中(§6.3 failure_detector)。
- **regime 止损**:`G_cycle` 切退潮 → 强制降仓/不接力。
- **时间/纪律止损**:次日设定止损纪律性执行(:642);退潮中端非核心 2B 尾盘漏单要走(:1292);龙头二连跌停次日不修复 → 证伪离场(:1291)。
- **否决权**:`G_risk` 可**否决** Co-pilot 的持有/买入建议(硬约束)。

**`[修P8]` 公告/监管 hard-constraint 否决器**(轮回一级禁忌,硬否决非软概率):减持就跑、辟谣就跑(:13–14);消息驱动题材竞价有龙一龙二不参与(:529);受制于监管的股(:702)。**优先级高于一切模式信号。**

**`[修P5]` fill-feasibility 模型**:论文动作恒可执行,交易"打板/一字/顶一字"常买不进(:865,:30)。建模封单量、排板位置、撤单风险(:16);**不可成交动作在 PRM 给中性/惩罚分**,在决策包给 `buyable=false`。

**硬守卫规则表(凌驾 Agent 建议)**:退潮不接力、不接弱转强 · 不上头计划交易 · 普遍/严重不及预期跑路 · 空仓修身(无模式不买)· 单票/单日仓位上限 · 相关性集中度上限 · 连亏熔断 · 涨停不可成交不挂单。

### 6.8 风控防过拟合专层(贯穿 outer loop)

集中四道工程闸(细节见 §8):**OOS/walk-forward 门 · 能力地板熔断 · frozen-doctrine 影子对照 · regime 漂移检测**。外加交易现实约束:T+1、最小变动、停牌、印花税/手续费、冲击成本。

### 6.9 评测体系(指标 / 消融 / Pareto / 技能-oracle 缺口)

**主指标(不止收益)**:
1. **决策质量分**(PRM 过程奖励均值);
2. 按 **模式 × regime** 的胜率 / 盈亏比 / 期望;
3. 风险调整收益(夏普 / 卡玛 / 最大回撤);
4. **成本后净收益 Pareto**(收益 vs 回撤&换手,↔ 论文 cost-vs-completion);
5. **技能-oracle 缺口**(§6.3 `oracle_gap`,↔ 论文 Fig8 Dijkstra deficit,直接度量技能自改进);
6. 覆盖率 / 择时命中率;
7. **自进化增益曲线**(随周期/迭代是否改进,↔ 论文 Fig7 里程碑推进)。

**`[修P13]` 消融条件(直接对标论文,含最致命的负面对照)**:
- **Hmin**(裸 LLM + 通用提示)· **Hexpert**(轮回.docx 原文人工填满、固定不进化)· **HCH**(from scratch 自建 / bootstrap frozen / bootstrap updating)。
- 验证自进化是否 recover 到 Hexpert 并超越;**bootstrap-updating 是否退化到比 frozen/Hmin 还差**(论文最重要负面发现:Red 上 updating 退化、子 Agent 继承率仅 6.4%)——**必须常驻 frozen-doctrine 影子组合,跑不赢即触发回退**(§8.3)。

**协议**:严格 OOS / walk-forward、**purged & embargoed CV** 防泄漏、多 seed、**regime 分层评测**、纸面 → 小资金实盘爬坡门槛。**在线监控**:实盘决策质量衰减预警、与回放分布的 KL 漂移。

## 7. 自进化双闭环:端到端串讲

**(1) 一个交易日(inner-loop 单步 + ACT)**
盘前 `G_cycle` 读 `o_t` 出 per-题材线 regime → 选可用技能集 → 编排器 master 分派建候选池 → 竞价预期差判读排序 → 仓位层定档 → 风控守卫过禁忌/监管/fill-feasibility → 出 `DecisionPackage` → **人确认/否决/改单** → 真实成交回流 `τ`。

**(2) 当日收盘(inner-loop Refiner,F=1)**
Refiner 读当日 `τ` → 识别失败签名 → 四遍 CRUD(Δp doctrine / ΔG 识别器 / ΔK 技能含孵化 / ΔM 教训)→ `H_{t+1}=H_t⊕Δ` → 明日生效。**不 reset。**

**(3) 一个周期轮回结束(相位切换大复盘)**
检测到"龙头与补涨龙共振下跌 + 旧核止跌新核反核"→ 大复盘:重写整段 doctrine、对本轮失效技能批量 retire→dormant、对本轮新生效模式 codify、把本轮具名案例写入 M。episode 内 doctrine 局部单调累积、跨 regime decay。

**(4) 跨周期(outer-loop 协同学习)**
积累若干周期的回放轨迹 → `π_θ` 在活精炼 H 内跑 K 步 rollout → PRM 过程奖励 → 低奖励窗口 teacher+oracle relabel(带 regime 标签)→ LoRA 更新 θ → 影子盘 A/B + OOS 门 → 达标则 θ_{k+1} 回灌,否则回滚。θ 与 H co-adapt。

## 8. 非平稳性专论:交易相对 Pokémon 多出来的难点与六道闸

Pokémon **平稳 + 完美固定 oracle(Dijkstra)**:迷宫不变,最优路径永远最优。交易把这两条全部打破,这是本架构与论文最本质的分歧。

**难点 1 · oracle 漂移**:已实现未来是过去 regime 的真理;2024 连板生态里完美的弱转强买点,2025 容量生态里可能是陷阱。
> **闸①+②**:**regime-aware relabel**——所有 oracle 标签、PRM 权重、K 统计、M 教训都打 regime 标签,训练按"目标 regime 相似度"加权采样,**禁止跨 regime 无差别 relabel**;**decay-weighted statistics**——K/M 统计用时间+regime 双衰减,旧的降权不删。

**难点 2 · 对抗反身性(alpha decay)**:打法被广泛套利后溢价消失甚至反转;系统自己的成功侵蚀自己的 edge。
> **闸③**:**alpha-decay 监控器**(独立子 Agent)盯每个现役 K/G 的近窗 OOS 绩效斜率,持续下行 → Refiner retire/降权;**拥挤度/反身性特征**入观测喂 PRM 和 p。

**难点 3 · 退役打法的"轮回"复活(交易优于 Pokémon)**:废弃迷宫路径不复活,退役打法会随生态轮转重新有效(旧核止跌新核反核)。
> **闸④**:retire≠delete,进 **dormant 池**(保 regime 指纹);regime 回归 → 自动 revive 候选 → 孵化锦标赛重新竞争(§6.3)。

**难点 4 · 能力地板回退**:论文铁律,地板以下精炼无法 bootstrap(Flash-Lite 所有 CH 变体不如 minimalist)。交易地板 = 数据/特征/模型/某 regime 样本量不足,此时自进化 curve-fit 噪声,比不进化更危险。
> **闸⑤**:**地板探测**——上 outer loop 前先验:OOS 上自进化 H 不能稳定优于 Hmin/Hexpert 静态基线 → 判地板以下,**冻结 outer loop(退回方案 B)**;**新 regime 冷启动**——样本数<阈值时该 regime 下 θ 进化暂停,仅靠 doctrine + 人类经验。

**难点 5 · 前视/过拟合(因有完美 oracle 而尤其致命)**:前视偏差是头号杀手,过拟合过去 regime 是第二。
> **闸⑥**:**point-in-time + as-of-date**(任何特征/relabel 只用决策时刻前信息,回放强制时序隔离)· **OOS 门**(θ_{k+1}/新 K/G 须在未训练时段显著优于才上线)· **影子盘灰度**(上实盘前纸面验证滑点/买不进/流动性折算后仍有 edge)· **`[修P3]` frozen-doctrine 影子对照**(常驻冻结基线,自进化跑不赢即回退)· **survivorship 校正**(M 具名范例配等量"看着像但失败了"的负例)· **`[修P15]` 涌现技能审计闸**(拦截利用数据泄漏的作弊技能)· **模拟成本 vs 实盘成本 diff 警报**。

**一句话**:Pokémon 只需"学会走迷宫";交易需"学会走一座**每天改图、且会因为你学会而改图、但偶尔把老图换回来**的迷宫"——所以本架构在论文双环之上强制叠加这六道闸,把论文的平稳假设替换成交易的非平稳现实。

## 10. 关键风险与开放问题(诚实清单)

1. **能力地板**:基础/本地模型不够强时,自进化产生负价值(论文铁证)。→ 闸⑤,且 Phase-1 先用强冻结模型验证。
2. **oracle 漂移 + 反身性**:最根本的非平稳风险,六道闸是缓解而非根除;edge 本身有半衰期。
3. **co-learning 未确立收敛**(论文也没有):永远以 OOS + 影子盘为准,设停训/回滚准则。
4. **容量与冲击成本**:超短策略容量有限,规模上去 edge 衰减;仓位层须含容量约束。
5. **监管 / 合规**:游资打法处监管敏感区;公告/监管否决器为硬闸,全程审计留痕;本系统定位 co-pilot、人确认,不自动下单。
6. **数据工程是隐形大头**:PIT、L2、龙虎榜历史、题材成分历史时点版本,数据治理质量直接决定地板高低。
7. **teacher 循环验证风险**:teacher 知识若全来自同一 substrate,需引入独立信息源(实际盘面 oracle 为主、LLM teacher 只做归因)。

| 内环 / 外环 | inner / outer loop | 单 episode 内在线自精炼 / 跨迭代协同学习 |
| 过程奖励模型 | **PRM** | `R(s,a,τ)∈[0,1]` 给每个决策 transition 打分 |
| 前沿教师 | frontier teacher | 强 LLM,把 oracle 结果归因成决策包+理由 |
| 神谕 | **oracle** | 完美标签来源;此处=**已实现的未来**(↔Dijkstra),**会漂移** |
| 无重置 | reset-free | episode/迭代间不重启,状态跨训练累积 |
| 能力地板 | capability floor | 自进化能 bootstrap 的最低模型/数据质量门槛 |
| 周期轮回 | episode / reincarnation | 冰点→主升→退潮→冰点 的完整情绪周期 |
| regime | regime | 当前生态/情绪/流动性宏观状态,**per-题材线 + 全局母状态** |
| 情绪值 | sentiment score | regime-relative 归一化的情绪量化(非绝对阈值) |
| 弱转强 | weak-to-strong (W2S) | 今日上板比昨日早 / 小幅高开拉回示强 |
| 退潮不接力 | no-relay in ebb tide | 退潮期禁止接力的纪律口诀 |
| 预期差 / 预期链 | expectation gap / ladder | 实际相对预期偏离;核按钮…百万封单一字 的档位链 |
| 葵花宝典 | playbook grid | 情绪值×指数开盘强弱 → 模式选择 的决策手册 |
| alpha 衰减 | alpha decay | 打法被套利失效;oracle 漂移之源 |
| 决策包 | DecisionPackage / `a_t` | co-pilot 结构化输出 |
| 未来函数 / 前视 | lookahead bias | 用了当时不可得的信息(头号杀手,严防) |
| 幸存者偏差 | survivorship bias | 只看活下来的票/打法 |
| point-in-time | PIT / as-of-date | 当时点可得数据快照,杜绝未来函数 |
| 影子盘 | shadow / paper trading | 不下真单的纸面跟踪验证 |
